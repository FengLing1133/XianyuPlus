第三节：API 接口设计

RESTful 风格 · JWT Bearer 鉴权 · 统一响应格式

统一响应体

{
  "code": 200,        // 业务状态码
  "message": "success",
  "data": { }         // 具体数据
}

认证模块 — /api/auth

POST/api/auth/register注册（用户名+密码+昵称）
POST/api/auth/login登录，返回JWT token，Redis缓存
GET/api/auth/info获取当前用户信息（需认证）
PUT/api/auth/password修改密码（需认证）

用户模块 — /api/user

GET/api/user/{id}用户详情（发布的商品、信誉等）
PUT/api/user/profile编辑个人资料（需认证）
POST/api/user/avatar上传头像（需认证）

商品模块 — /api/product

GET/api/product商品列表（分页+搜索+分类+价格排序）
GET/api/product/{id}商品详情（含图片列表、卖家信息）
POST/api/product发布商品（需认证，含图片上传）
PUT/api/product/{id}编辑商品（仅发布者本人）
PUT/api/product/{id}/status下架/重新上架（仅发布者）
GET/api/product/my我的发布列表（需认证）

收藏 + 订单模块

POST/api/favorite/{productId}收藏/取消收藏切换（需认证）
GET/api/favorite我的收藏列表（需认证）
POST/api/order创建订单/购买（需认证）
GET/api/order?type=buy|sell我买到的/我卖出的（需认证）
PUT/api/order/{id}/status更新订单状态（需认证）

文件 + 聊天 + 管理后台

POST/api/file/upload上传图片，返回URL
ws/ws/chat/{userId}WebSocket 连接点（需token参数）
GET/api/message/{userId}获取与某用户的历史消息
GET/api/message/unread未读消息数
—/api/admin/**管理接口（需 admin 角色），含用户管理、商品审核、数据概览
GET/api/category分类列表（树形）

认证说明

所有 需认证 接口在请求头携带 Authorization: Bearer <token>，后端通过 Spring Security 过滤器统一校验。 匿名接口：登录、注册、商品列表、商品详情、分类列表。
