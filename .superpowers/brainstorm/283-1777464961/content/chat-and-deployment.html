第五节：聊天系统与部署方案

WebSocket + Redis | 开发环境 | 项目启动方式

聊天系统架构
Vue 前端 — ws:// — Spring WebSocket — pub/sub — Redis ←— MySQL

消息流转：发送 → Redis Pub/Sub 广播 → 接收方在线则实时推送 → 同时持久化到 MySQL

离线消息：接收方不在线时消息已存入 MySQL，上线后拉取未读消息

聊天核心逻辑

环节实现
连接鉴权WebSocket 握手阶段，从 URL 参数或首次消息中校验 JWT token
在线状态用户连接时 Redis 记录 userId → sessionId 映射
消息发送接收消息 → 存 MySQL → 通过 Redis Pub/Sub 推送给目标用户
历史消息HTTP REST 接口拉取：GET /api/message/{userId}，分页返回
未读计数前端定时轮询，同时 WebSocket 也可推送未读更新事件

项目启动方式

组件启动方式
MySQL本地安装，初始化 SQL 脚本，创建数据库 xianyu_plus
Redis本地安装，默认端口 6379
后端IDEA 打开 Maven 项目，运行主启动类，端口 8080
前端npm install && npm run dev，Vite 开发服务器端口 5173，代理 /api 到 8080

非功能设计说明

方面方案
分页MyBatis-Plus PageHelper，统一 PageResult<T> 返回
异常处理@RestControllerAdvice 全局捕获，统一返回 {code, message}
ID 生成MyBatis-Plus 雪花算法（ASSIGN_ID）
跨域Spring CORS 配置允许 localhost:5173
密码加密BCryptPasswordEncoder
