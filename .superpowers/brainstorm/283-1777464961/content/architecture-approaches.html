项目架构方案对比

Vue 3 + Spring Boot + MySQL | 完整版功能

A

单体单模块 — 简洁实用

单个 Spring Boot 项目 + 单个 Vue 项目，所有功能在一个模块内按包分层（controller/service/mapper）

适合：快速开发，逻辑集中，调试方便

优点

  • 项目结构简单清晰
  • 开发调试快速
  • 无需处理模块间依赖

缺点

  • 代码量增大后难以维护
  • 模块边界模糊
B

多模块分层 — 工程化推荐

Maven 多模块：common（公共）、framework（框架）、service（业务）、chat（聊天）分离；前端 Pinia 状态管理

适合：毕设展示工程能力，模块职责清晰

优点

  • 模块职责清晰
  • 利于答辩展示架构
  • websocket 独立部署友好

缺点

  • 多模块配置稍复杂
  • 需要管理模块间依赖
C

前后端全部拆分 — 过度设计

后端拆分为多个独立服务；前端拆分为用户端 + 管理端两个独立项目

适合：大型生产项目，但对毕设来说成本过高

优点

  • 极致解耦

缺点

  • 开发和部署复杂度高
  • 毕设场景过度设计
  • 时间成本大
