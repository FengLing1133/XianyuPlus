第四节：前端结构与路由设计

Vue 3 + Element Plus + Pinia + Vue Router

页面结构

用户端
🏠 首页（商品瀑布流 + 搜索 + 分类导航）
📋 商品详情页（图片轮播 + 卖家信息 + 立即购买/联系卖家）
✏️ 发布商品页（表单 + 图片上传）
💬 聊天页（会话列表 + 聊天窗口，WebSocket实时）
❤️ 我的收藏
📦 订单管理（我买到的 / 我卖出的）
👤 个人中心（资料编辑 + 我的发布）
🔑 登录/注册
管理端（路由守卫分离）
📊 仪表盘（用户数、商品数、订单数统计卡片）
👥 用户管理（列表 + 封禁/解封）
📦 商品管理（列表 + 审核 + 下架）
📋 订单管理（全部订单查看）

路由表

路径页面权限
/首页（商品列表）公开
/product/:id商品详情公开
/login登录仅未登录
/register注册仅未登录
/publish发布商品需登录
/publish/:id编辑商品需登录
/chat聊天（会话列表）需登录
/chat/:userId聊天窗口需登录
/favorite我的收藏需登录
/order订单管理需登录
/profile个人中心需登录
/admin/*管理后台admin角色

前端项目结构

src/
├── api/           # axios 封装 + 各模块 API 调用
│   ├── request.js      # 拦截器（JWT注入、401处理）
│   ├── auth.js         # 登录/注册
│   ├── product.js      # 商品相关
│   ├── order.js        # 订单
│   └── message.js      # 聊天
├── stores/        # Pinia 状态管理
│   ├── user.js         # 用户信息 + token
│   ├── chat.js         # WebSocket连接 + 消息
│   └── notification.js # 未读消息数（轮询/WS推送）
├── views/         # 页面组件
├── components/    # 公共组件（ProductCard, ImageUploader等）
├── router/        # 路由配置 + 守卫
└── utils/         # 工具函数

核心组件的设计取舍

组件实现方式
图片上传el-upload + 本地预览 + 拖拽排序，最多9张
商品卡片瀑布流布局（CSS Grid）+ 懒加载图片
聊天窗口WebSocket 连接管理 + 消息气泡 + 滚动到底部
路由守卫beforeEach 检查 token 存在 + role 校验
状态持久化pinia-plugin-persistedstate 将 token/user 持久化到 localStorage
